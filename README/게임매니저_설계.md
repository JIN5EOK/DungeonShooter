# 임시작성중...

# 개요
> 인게임 관리 클래스들에 대한 설계 문서입니다


- 핵심 포인트
  - 플레이어 관련 데이터가 플레이어 게임오브젝트에 종속되어서는 안된다!
  - 그렇게 되면 게임이 완전히 끝날 때 까지 플레이어 게임오브젝트를 파괴시키거나 다른 게임오브젝트로 교체하는게 불가능해진다.
- 클래스 구상
- `GameManager`
  - 게임 스테이지의 시작(스테이지 생성)과 끝을 담당(게임 종료)
  - 그 외 인게임 내 슈퍼클래스의 역할이 필요하다면 담당
- `TimeScaleManager`
  - 게임의 일시정지 여부를 결정
  - 레퍼런스 카운트 방식으로 설계, 일시정지가 필요한 객체들은 일시정지를 요청하고 필요없어지면 해제 요청, 현재 요청 갯수가 0이 되어야 일시정지가 풀리도록 함
- `PlayerInstanceManager`
  - 역할
    - 플레이어 게임오브젝트 & 플레이어를 이루는 스킬,스탯,인벤토리,Hp,경험치 등..의 데이터를 연동시킴
- `ExpManager`
  - 역할
    - 경험치 누적, 경험치가 차면 레벨업 이벤트 실행
- `PlayerSkillController`
  - 역할
    - 플레이어 고유 스킬 관리 및 스킬 레벨업 처리 담당
      - `EntitySkillGroup`에 스킬을 등록할 수 있지만 등록만 할 뿐 상세 로직은 들어가지 않음 (어떤게 플레이어의 고유 액티브 스킬인지 구분, 스킬 레벨업 기능 등)
    - 레벨업이 가능한 스킬들을 소유하고 스킬 레벨업시 `EntitySkillGroup`에 등록된 스킬 교체처리
    - 단축키로 사용 가능한 액티브 스킬들을 관리하고 사용 가능하도록 도와줌
